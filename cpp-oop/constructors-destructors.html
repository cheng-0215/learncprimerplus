构造函数与析构函数 ​

构造函数和析构函数是C++中类的特殊成员函数，它们在对象的生命周期中扮演着重要角色。本章将详细介绍构造函数和析构函数的概念、种类和使用方法。

构造函数 ​

构造函数是一种特殊的成员函数，在创建对象时自动调用。它的主要作用是初始化对象的数据成员，确保对象在创建时处于有效状态。

构造函数的特点 ​

  • 构造函数的名称与类名相同
  • 构造函数没有返回类型，甚至不能指定void
  • 构造函数可以有参数，也可以没有参数
  • 一个类可以有多个构造函数（函数重载）

默认构造函数 ​

默认构造函数是没有参数的构造函数，或者所有参数都有默认值的构造函数。

cpp
class Box {
private:
    double length;
    double width;
    double height;
    
public:
    // 默认构造函数
    Box() {
        length = 0.0;
        width = 0.0;
        height = 0.0;
    }
};

// 使用默认构造函数创建对象
Box box1; // 所有维度都被初始化为0.0

如果没有定义任何构造函数，编译器会自动生成一个默认构造函数。但是，一旦定义了任何构造函数，编译器就不会再生成默认构造函数。

带参数的构造函数 ​

带参数的构造函数允许在创建对象时提供初始值。

cpp
class Box {
private:
    double length;
    double width;
    double height;
    
public:
    // 带参数的构造函数
    Box(double l, double w, double h) {
        length = l;
        width = w;
        height = h;
    }
};

// 使用带参数的构造函数创建对象
Box box2(10.0, 5.0, 2.0);

构造函数重载 ​

一个类可以有多个构造函数，只要它们的参数列表不同。

cpp
class Box {
private:
    double length;
    double width;
    double height;
    
public:
    // 默认构造函数
    Box() {
        length = width = height = 0.0;
    }
    
    // 带一个参数的构造函数（所有维度相同）
    Box(double side) {
        length = width = height = side;
    }
    
    // 带三个参数的构造函数
    Box(double l, double w, double h) {
        length = l;
        width = w;
        height = h;
    }
};

// 使用不同的构造函数
Box box1;               // 调用默认构造函数
Box box2(5.0);          // 调用带一个参数的构造函数
Box box3(10.0, 5.0, 2.0); // 调用带三个参数的构造函数

初始化列表 ​

构造函数可以使用初始化列表来初始化数据成员，这通常比在构造函数体内赋值更高效。

cpp
class Box {
private:
    double length;
    double width;
    double height;
    
public:
    // 使用初始化列表的构造函数
    Box(double l, double w, double h)
        : length(l), width(w), height(h) {
        // 构造函数体可以为空，或者包含其他初始化代码
    }
};

初始化列表对于常量成员和引用成员的初始化是必需的，因为它们必须在创建时初始化。

cpp
class Example {
private:
    const int constMember;
    int& refMember;
    
public:
    // 必须使用初始化列表初始化常量和引用成员
    Example(int value, int& ref)
        : constMember(value), refMember(ref) {
    }
};

委托构造函数（C++11） ​

C++11引入了委托构造函数，允许一个构造函数调用同一个类的另一个构造函数。

cpp
class Box {
private:
    double length;
    double width;
    double height;
    
public:
    // 主构造函数
    Box(double l, double w, double h)
        : length(l), width(w), height(h) {
    }
    
    // 委托给主构造函数
    Box()
        : Box(0.0, 0.0, 0.0) {
    }
    
    // 也委托给主构造函数
    Box(double side)
        : Box(side, side, side) {
    }
};

析构函数 ​

析构函数是一种特殊的成员函数，在对象销毁时自动调用。它的主要作用是释放对象占用的资源，执行必要的清理工作。

析构函数的特点 ​

  • 析构函数的名称是类名前面加上波浪号（~）
  • 析构函数没有返回类型，也没有参数
  • 一个类只能有一个析构函数
  • 如果没有定义析构函数，编译器会生成一个默认的析构函数

析构函数的定义和使用 ​

cpp
class Resource {
private:
    int* data;
    
public:
    // 构造函数
    Resource() {
        data = new int[100]; // 分配动态内存
        cout << "Resource acquired" << endl;
    }
    
    // 析构函数
    ~Resource() {
        delete[] data; // 释放动态内存
        cout << "Resource released" << endl;
    }
};

void useResource() {
    Resource res; // 构造函数被调用
    // 使用资源...
} // 函数结束时，res超出作用域，析构函数被调用

析构函数的调用时机 ​

析构函数在以下情况下被调用：

  1. 局部对象超出作用域时
  2. 使用delete删除动态分配的对象时
  3. 程序结束时，全局对象被销毁
  4. 临时对象的生命周期结束时
cpp
int main() {
    // 1. 局部对象
    {
        Resource localRes; // 构造函数被调用
    } // 作用域结束，析构函数被调用
    
    // 2. 动态分配的对象
    Resource* pRes = new Resource(); // 构造函数被调用
    delete pRes; // 析构函数被调用
    
    return 0;
} // 3. 程序结束，全局对象的析构函数被调用

构造函数和析构函数的重要性 ​

RAII（资源获取即初始化） ​

RAII是C++中的一种编程技术，它将资源的生命周期与对象的生命周期绑定在一起。通过在构造函数中获取资源，在析构函数中释放资源，可以确保资源的正确管理，避免资源泄漏。

cpp
class FileHandler {
private:
    FILE* file;
    
public:
    // 构造函数打开文件
    FileHandler(const char* filename, const char* mode) {
        file = fopen(filename, mode);
        if (!file) {
            throw std::runtime_error("Failed to open file");
        }
    }
    
    // 析构函数关闭文件
    ~FileHandler() {
        if (file) {
            fclose(file);
        }
    }
    
    // 文件操作方法
    void write(const char* data) {
        if (file) {
            fputs(data, file);
        }
    }
};

void processFile() {
    try {
        FileHandler handler("example.txt", "w");
        handler.write("Hello, World!");
        // 即使这里发生异常，文件也会被正确关闭
    } catch (const std::exception& e) {
        std::cerr << e.what() << std::endl;
    }
} // handler超出作用域，析构函数被调用，文件被关闭

防止资源泄漏 ​

构造函数和析构函数的正确使用可以防止资源泄漏，特别是在异常发生时。

cpp
class MemoryResource {
private:
    int* buffer1;
    int* buffer2;
    
public:
    MemoryResource(size_t size) {
        buffer1 = new int[size]; // 分配第一个缓冲区
        
        try {
            // 可能抛出异常的操作
            buffer2 = new int[size * 10]; // 分配第二个缓冲区
        } catch (...) {
            // 清理已分配的资源
            delete[] buffer1;
            throw; // 重新抛出异常
        }
    }
    
    ~MemoryResource() {
        delete[] buffer1;
        delete[] buffer2;
    }
};

总结 ​

  • 构造函数在对象创建时自动调用，用于初始化对象
  • 析构函数在对象销毁时自动调用，用于释放资源
  • 一个类可以有多个构造函数，但只能有一个析构函数
  • 构造函数和析构函数是实现RAII（资源获取即初始化）的关键
  • 正确使用构造函数和析构函数可以防止资源泄漏，提高程序的健壮性
